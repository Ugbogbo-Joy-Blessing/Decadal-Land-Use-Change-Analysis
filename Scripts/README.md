# 🌍 Spatio-Temporal LULC Analysis of Ikeja LGA

## 📌 Repository Overview
This repository contains a suite of Google Earth Engine (GEE) JavaScript codebase designed for advanced Land Use and Land Cover (LULC) analysis in **Ikeja Local Government Area** (Lagos State, Nigeria). 

Ikeja is a highly saturated, spectrally complex urban core. This repository demonstrates the application of machine learning algorithms to accurately map this urban matrix, track annual changes in urban green infrastructure, and mathematically quantify spatial transitions (e.g., urban infill vs. canopy maturation) over a decadal epoch (2016–2026).

## 🗂️ Included Scripts

### 1. `LULC_Annual_Series_2016_2026.js` (Classification)
This script automates the annual classification of LULC for any target year. 
* **Algorithm:** Supervised Random Forest (`ee.Classifier.smileRandomForest`).
* **Feature Engineering:** Computes and appends critical spectral indices (NDVI, NDBI) to overcome the "Mixed Pixel Effect" common in 30m resolution urban data.
* **Validation:** Automatically splits ground-truth data (70/30) and prints a formal Confusion Matrix, Overall Accuracy, and Kappa Coefficient to the GEE console.

### 2. `Ikeja_Change_Detection.js` (Spatial Statistics)
This script performs rigorous spatio-temporal change detection across the LGA to quantify urban dynamics between two epochs (2016 and 2026).
* **Methodology:** Rather than just identifying net area changes, this script logically combines the two categorical maps to compute a mathematical **"From-To" Transition Matrix**.
* **Outputs:** Accurately calculates the exact area (in km²) of specific transition phenomena (e.g., Built-up transitioning to Vegetation, indicating mature tree canopy growth over residential zones).

## ⚙️ Data Sources & Platform
* **Platform:** Google Earth Engine (GEE) Cloud Computing Architecture.
* **Imagery:** Landsat 8 (OLI/TIRS) Collection 2, Tier 1 Level-2 (Surface Reflectance). *Pre-processed for atmospheric and geometric corrections.*
* **Boundary:** FAO/GAUL Level-2 boundaries (filtered to Ikeja).

## 🚀 How to Execute
1. Copy the contents of either script into the [Google Earth Engine Code Editor](https://code.earthengine.google.com/).
2. Run `Ikeja_Annual_LULC.js` to generate the baseline categorical maps and export the `.tif` files to your Google Drive via the **Tasks** tab.
3. To execute the transition analysis, run `Ikeja_Change_Detection.js`, ensuring it is pointed to your previously classified Image Assets. The exact transition areas (in km²) will print directly to the console.

## 📊 Cartographic Outputs
The `.tif` exports generated by these scripts are designed to be imported into ArcGIS Pro or QGIS for final cartographic layout (applying standard LULC symbology, legends, and neatlines).
